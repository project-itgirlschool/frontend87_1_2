<!-- При работе в одной ветке в GitHub с другими пользователями, перед тем как вводить команду git push, рекомендуется сначала выполнить команду git pull. Это необходимо для того, чтобы синхронизировать локальный репозиторий с удалённым и избежать конфликтов:
- внесите необходимые изменения в файлы,
- добавьте измененные файлы git add,
- зафиксируйте изменения git commit -m"#",
- выполните git pull, чтобы получить последние изменения из удалённого репозитория (если кто-то параллельно с вами работал и внес изменения),
- если при выполнении git pull возникли конфликты, разрешите их вручную в предлагаемом окне (Accept Curent Change - принять текущее изменение/ Accept Incoming Change - принять входящее (от другого участника проекта) изменение/ Accept Incoming Change - принять оба изменения/,
- после разрешения конфликтов зафиксируйте изменения git add, git commit -m "Разрешены конфликты",
- отправка ваших изменений в удалённый репозиторий git push. -->
